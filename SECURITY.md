# Security Policy

MARROW is positioned around trust, auditability, and sovereignty. Security reports are taken seriously.

## Reporting a vulnerability

**Please do not open a public issue for security vulnerabilities.**

Report privately to **ops@elevenworks.io**, or use GitHub’s **private vulnerability reporting** (Security → Report a vulnerability) on this repository.

Please include:

- a description of the issue and its impact,
- steps to reproduce or a proof of concept,
- affected version/commit, and any suggested remediation.

## What to expect

- Acknowledgement of your report within a few business days.
- An honest assessment and, where valid, a fix timeline.
- Credit for responsible disclosure if you’d like it.

## Scope

This project is at a foundational stage; the substrate and its surfaces are evolving. Reports about the event-sourced core, authentication/authorization, tenant isolation, agent action boundaries, and data-residency guarantees are especially welcome.
